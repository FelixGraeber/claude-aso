# Keyword Placement Rules by Platform

## iOS App Store — Keyword Weight by Field

| Priority | Field | Weight | Notes |
|----------|-------|--------|-------|
| 1 | App Name | Highest | Most impactful placement. Include primary keyword. |
| 2 | Subtitle | High | Second-most impactful. Complementary keywords. |
| 3 | Keywords Field | Medium | Unique terms only — don't repeat name/subtitle words. |
| 4 | IAP Display Names | Low | Bonus keyword coverage. |
| — | Description | None | NOT indexed. Don't optimize for keywords. |
| — | Promotional Text | None | NOT indexed. |

### iOS Placement Strategy

1. **Primary keyword** → App Name (must appear here)
2. **Secondary keyword** → Subtitle
3. **Long-tail / synonyms / related** → Keywords field
4. **Never duplicate** words across Name, Subtitle, and Keywords — Apple indexes all three independently

### iOS Keywords Field Rules
- Comma-separated, NO spaces after commas: `word1,word2,word3`
- Use all 100 characters
- Singular forms only (Apple auto-matches plurals)
- Omit: articles (a, the), prepositions (for, with), your app name (already indexed)
- No competitor brand names (policy violation)

## Google Play — Keyword Weight by Field

| Priority | Field | Weight | Notes |
|----------|-------|--------|-------|
| 1 | Title | Highest | Include primary keyword naturally. |
| 2 | Short Description | High | Secondary keyword + benefit. |
| 3 | Full Description | High | 2-3% density for primary. Distribute naturally. |
| 4 | Developer Name | Low | Minor signal. |
| 5 | User Reviews | Medium | Review text is indexed. Can't directly control. |
| — | What's New | None | NOT indexed. |

### Android Placement Strategy

1. **Primary keyword** → Title (first position preferred)
2. **Primary keyword** → First sentence of Full Description
3. **Secondary keywords** → Short Description
4. **All keywords** → Distributed through Full Description at 1-3% density
5. **Backlinks** → External links with keyword anchor text boost rankings

### Android Description Structure

```
[Opening paragraph — primary keyword in first sentence, value proposition]

[Feature section 1 — secondary keyword naturally integrated]
[Feature section 2 — additional keywords]
[Feature section 3 — use case keywords]

[Social proof — awards, press mentions, user count]

[CTA — download prompt, key differentiator]
```

### Android Density Guidelines

| Keyword Type | Target Density | Max Mentions (in 4000 chars) |
|-------------|----------------|------------------------------|
| Primary | 2-3% | 8-12 |
| Secondary | 1-2% | 4-8 |
| Long-tail | 0.5-1% | 2-4 |
| Any single keyword | ≤3% | ≤12 |

## Cross-Platform Rules (Apply to Both)

1. **No keyword stuffing** — >3 repetitions of exact same phrase = stuffing
2. **Natural language** — both algorithms increasingly reward natural, readable text
3. **User intent** — match keywords to what users actually search for, not marketing jargon
4. **Competitor names** — NEVER use competitor brand names as keywords (policy violation on both)
5. **Exact match declining** — both platforms use semantic/NLP matching increasingly
